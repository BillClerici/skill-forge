Hybrid Architecture: Traditional Services + AI Agents

Core Principle: Use traditional microservices for CRUD, security, and deterministic logic. Use AI agents for reasoning, creativity, and complex decision-making.

AI Agents (Pink Components)

  • AI Game Master Agent: Core narrative experience. Autonomous storytelling with tool use.
  • Content Adaptation Agent: Intelligently modifies Worlds for different Universes.
  • Educational Assessment Agent: Maps player actions to learning outcomes.
  • Parental Monitoring Agent: Analyzes patterns, generates summaries.
  • Universe Recommendation Agent: Interviews users, explains recommendations.
  • NPC Conversation Agents: Per-NPC personality with memory (major NPCs only).

Traditional Microservices (Green/Blue/Teal)

  • All CRUD operations: Accounts, Characters, Campaigns, Sessions
  • Security-critical: Auth, Time Limits, Content Filtering, Billing
  • Deterministic logic: Cognitive skills, campaign state, relationships
  • High-frequency ops: Session management, real-time updates

What AI Agents CANNOT Do

  • Override parental controls or content filters (rule-based enforcement)
  • Make billing/payment decisions (deterministic only)
  • Decide time limits (hard limits, agents only notify)
  • Handle auth/authorization (security-critical)
  • Directly modify databases (must call microservices)

Implementation Roadmap

Phase 1: Foundation (Months 1-3)

  • Build traditional microservices (Account, Character, Universe, World)
  • Implement PostgreSQL, DocumentDB, Redis data layer
  • Deploy API Gateway and authentication

Phase 2: MCP Infrastructure (Months 3-4)

  • Build Player Data, World/Universe, Educational Standards MCP servers
  • Implement MCP authentication and rate limiting

Phase 3: AI Game Master (Months 4-6)

  • Develop AI Game Master Agent with LangGraph
  • Integrate Claude API, connect to MCP servers
  • Implement tool calling, conversation persistence
  • Deploy orchestration with cold start and cost tracking
  • LAUNCH BETA: Game Master with 1-2 Universes, 2-3 Worlds

Phase 4: Additional Agents (Months 6-9)

  • Deploy Content Adaptation, Educational Assessment, Recommendation agents
  • Implement quality monitoring and human review queues

Phase 5: Scale & Optimize (Months 9-12+)

  • Deploy Parental Monitoring, NPC Conversation agents
  • Optimize costs (caching, model selection, prompt engineering)
  • Expand warm agent pool, implement advanced A/B testing

AI Agent Cost Projections

Assumptions

  • Claude Sonnet: $3 input / $15 output per million tokens
  • Average session: 30 min, ~50K tokens (with caching: $0.10/session)
  • Caching reduces costs by 30%

Monthly Cost by User Base

  • 1,000 users (10 sessions/month): $1,000/month
  • 10,000 users: $10,000/month
  • 100,000 users: $100,000/month
  • 1,000,000 users: $1,000,000/month

Revenue Coverage

  • Individual ($10/mo): 100 sessions → AI cost covered if < $0.10/session
  • Family ($25/mo): 250 sessions → AI cost covered
  • Educational ($50/mo): 500 sessions → AI cost covered
  • Margins: 50-70% of revenue after AI costs (at scale)
